# Kubernetes Pods – Base Image Variations (Projects 1–10)

## Objective
Deploy Kubernetes Pods using different base images to understand:
- Container startup behavior
- Runtime differences
- Image size & resource impact
- Debugging patterns

## Images Covered
- nginx
- apache
- python
- node
- go
- ruby
- php
- java
- rust
- dotnet

## Common Commands
kubectl apply -f pod.yaml  
kubectl get pods -o wide  
kubectl describe pod <pod-name>  
kubectl logs <pod-name>

## Real-World Relevance
Understanding base images is critical for:
- Image optimization
- Security hardening
- Faster startup times
- Production troubleshooting
